Like many math-wielding civilizations, the ancient Egyptians had a need to represent fractional values. Their representation of fractions was similar to our current system, with one limitation: the numerator (top part) of the fraction is always 1. That is, they could only write fractions like these:

1/2, 1/3, 1/4, 1/5, 1/6, ...

Any other fractional amount was represented as the sum of these simple fractions. Note that in the Egyptian fraction system, each denominator can only be used once.

3/5 = 1/2 + 1/10  (NOT 1/5 + 1/5 + 1/5)

Any modern fraction with a positive integral numerator and denominator can be represented in the Egyptian style. Your task is to write a program that accepts a "modern" fraction and outputs the Egyptian representation. Note that there may be mulitple valid representations. (Example: 1/2 could be written "1/3 + 1/6"). Your program should always use the largest unit possible, and display the components in descending order. Example:

4/7 = 1/2 + 1/14              (CORRECT)
4/7 = 1/3 + 1/6 + 1/14        (INCORRECT)

3/4 = 1/2 + 1/4               (CORRECT)
3/4 = 1/4 + 1/2               (INCORRECT)
3/4 = 1/2 + 1/5 + 1/20        (INCORRECT)
3/4 = 1/3 + 1/5 + 1/6 + 1/20  (INCORRECT)

Input

Each line of input will be two positive non-zero integers, the first less than or equal to the second, representing a fraction in (0, 1].

Output

For each input, print a line with the Egyptian denominators in ascending order.

Example

STDIN:

3 5
5 7
5 13

STDOUT:

2 10
2 5 70
3 20 780
